Let Typescript do all types of references for you.\
Remove most of your self-written types and improve the readability of the code.

> TypeScript provides several utility types to facilitate common type transformations.
> These utilities are available globally.


[Typescript documentation](https://www.typescriptlang.org/docs/handbook/utility-types.html)

## `Omit<Type, Keys>`

> Constructs a type by picking all properties from Type and then removing Keys (**string literal** or union of **string literals**).

```typescript
interface User {
  name: string;
  twitter: string;
  discord: string;
}

//omit string literal
interface SocialPlatforms = Omit<User, 'name'>; // will look like this ðŸ‘‡

type SocialPlatforms = {
  twitter: string;
  discord: string;
};

//omit union of string literals
type Name = Omit<User, 'twitter' | 'discord'>;// will look like this ðŸ‘‡

type UserName = {
  name: string;
};
```

## `Pick<Type, Keys>`

> Constructs a type by picking the set of properties Keys (**string literal** or union of **string literals**) from Type.

```typescript
interface Post {
  _id: string;
  title: string;
  publishedAt: string;
  description: string;
  body: MDX;
  slug: string;
  readingTime: string;
}

type PostSummary = Pick<Post, 'title' | 'description'>; // will look like this ðŸ‘‡

type PostSummary = {
  title: string;
  description: string;
};
```

## `Partial<Type>`

> Constructs a type with all properties of Type set to optional. This utility will return a type that represents all subsets of a given type.

```typescript
interface User {
  name: string;
  email:string;
}


type UserPartial =  = Partial<User>; // will look like this ðŸ‘‡

type User {
  name?: string;
  email?:string;
}
```
---

## `Required<Type>`

> Constructs a type consisting of all properties of Type set to required. The opposite of Partial.

```typescript
interface UserPartial {
  name?: string;
  email?:string;
}

type UserRequired =  = Partial<User>; // will look like this ðŸ‘‡

type User {
  name: string;
  email:string;
}
```

## `Record<Keys, Type>`

> Constructs an object type whose property keys are Keys and whose property values are Type.
This utility can be used to map the properties of a type to another type.

```typescript
interface UserInfo {
  email: string;
  dob: string;
}

type Name = 'alex' | 'mark' | 'adele';

const users: Record<Name, UserInfo> = {
  alex: { email: 'alex@email.com', dob: '08/07/1984' },
  mark: { email: 'mark@email.com', dob: '07/31/1992' },
  adele: { email: 'adele@email.com', dob: '10/23/1996' },
};

users.mark; //will be the type of ðŸ‘‡

const users: Record<Name, UserInfo>;
```

## `Parameters<Type>`

> Constructs a tuple type from the types used in the parameters of a function type Type.

```typescript
function f(arg: { a: number; b: string }) {
  return arg.a
}

type Params = Parameters<typeof f>; //will look like ðŸ‘‡

type Params = [
  arg: {
    a: number;
    b: string;
  }
];
```
---


## `ReturnType<Type>`

> Constructs a type consisting of the return type of function Type.

```typescript
function f(arg: { a: number; b: string }) {
  return arg.a
}

type ReturnTypeFromF= ReturnType<typeof f>; //will look like ðŸ‘‡

type ReturnTypeFromF = number
```
